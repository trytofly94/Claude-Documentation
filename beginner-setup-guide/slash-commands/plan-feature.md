Erstelle einen detaillierten Plan für das folgende Feature: $ARGUMENTS

## Phase 1: Verstehen & Analysieren

1. **Anforderungen klären**
   - Was soll das Feature genau tun?
   - Welche User Stories gibt es?
   - Welche Akzeptanzkriterien müssen erfüllt sein?

2. **Bestehenden Code analysieren**
   - Welche Dateien sind relevant?
   - Welche Architektur-Pattern werden genutzt?
   - Gibt es ähnliche Features, an denen ich mich orientieren kann?

3. **Dependencies identifizieren**
   - Welche externen Packages werden benötigt?
   - Welche internen Module müssen angepasst werden?

## Phase 2: Design

1. **Architektur-Entscheidungen**
   - Welche Design Patterns passen?
   - Wie fügt sich das Feature in die bestehende Architektur ein?
   - Welche Interfaces/APIs werden benötigt?

2. **Datenmodell**
   - Welche Datenstrukturen werden benötigt?
   - Brauchen wir neue Datenbank-Tabellen/Modelle?
   - Wie sehen Relationships aus?

3. **API Design** (falls relevant)
   - Welche Endpoints werden benötigt?
   - Request/Response Struktur?
   - HTTP Methods & Status Codes?

## Phase 3: Implementierungs-Plan

1. **Schritt-für-Schritt Vorgehen**
   Erstelle eine nummerierte Liste aller Implementierungs-Schritte:
   - [ ] Schritt 1: ...
   - [ ] Schritt 2: ...
   - [ ] etc.

2. **Tests planen**
   - Welche Unit Tests sind nötig?
   - Welche Integration Tests?
   - Welche Edge Cases müssen getestet werden?

3. **Security Considerations**
   - Welche Security-Aspekte müssen beachtet werden?
   - Input Validation?
   - Authentication/Authorization?
   - Potenzielle Vulnerabilities?

## Phase 4: Testing & Quality

1. **Test Strategy**
   - Test-Driven Development (Tests erst)?
   - Welche Test-Frameworks nutzen wir?
   - Coverage-Ziele?

2. **Code Quality**
   - Welche Coding Standards gelten?
   - Linting/Formatting?
   - Dokumentation?

## Phase 5: Deployment

1. **Environment Variables**
   - Welche neuen Env Vars sind nötig?

2. **Migrations** (falls relevant)
   - Datenbank-Migrations nötig?
   - Backwards Compatibility?

3. **Rollout Plan**
   - Feature Flags?
   - Gradual Rollout?

## Zusammenfassung

Erstelle am Ende eine **Executive Summary**:
- **Was:** Kurzbeschreibung des Features
- **Warum:** Business Value
- **Wie:** High-level Approach
- **Wann:** Geschätzte Complexity (Small/Medium/Large)
- **Risiken:** Potenzielle Probleme
